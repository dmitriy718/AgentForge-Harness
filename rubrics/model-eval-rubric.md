# Model Evaluation Rubric

Use for local and hosted model comparisons.

## Scores

0: failed or unsafe  
1: partially useful but unreliable  
2: useful with minor correction  
3: production-useful for this workload

## Categories

- Correctness
- Grounding
- Tool/command realism
- Constraint handling
- Refusal/privacy judgment
- Structured output quality
- Latency
- Resource use

## Decision Rule

The winning model must be good enough on the target workload. Do not choose a model because it wins an unrelated benchmark.
