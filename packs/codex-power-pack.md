# Codex Power Pack

Use these clauses to make Codex work sharper.

## Implementation Clause

```text
Read the code first. Identify the local pattern. Make the smallest coherent change. Run targeted validation. Do not modify unrelated files.
```

## Diagnostic Clause

```text
Do not guess the cause. Rank hypotheses by evidence, run the smallest test that distinguishes them, and only then propose a fix.
```

## Review Clause

```text
Findings first. Every finding needs file/line evidence, impact, and fix. No style-only comments unless they hide real risk.
```

## System Clause

```text
Before changing system state, show current state, intended change, validation, and rollback. Do not restart or delete without explicit intent.
```
