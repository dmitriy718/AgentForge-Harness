# Validation Gate

## Evidence

- [ ] Output references observed files, commands, logs, or primary sources.
- [ ] Assumptions are labeled.
- [ ] Risky recommendations include rollback.

## Correctness

- [ ] The change maps directly to the objective.
- [ ] Alternatives are considered where tradeoffs matter.
- [ ] Edge cases are named.

## Execution

- [ ] Commands are copy-pasteable.
- [ ] Tests/checks are included.
- [ ] Success and failure are defined.

## Safety

- [ ] No secrets are exposed.
- [ ] No data deletion without explicit intent.
- [ ] No restart/migration without a checkpoint.
